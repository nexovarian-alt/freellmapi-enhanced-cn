Windows Docker Desktop 安装教程

准备条件

安装并启动 Docker Desktop；准备一个专用文件夹。

生成加密密钥

$bytes = New-Object byte[] 32
$rng = New-Object System.Security.Cryptography.RNGCryptoServiceProvider
$rng.GetBytes($bytes); $rng.Dispose()
$key = -join ($bytes | ForEach-Object { $_.ToString('x2') })
"ENCRYPTION_KEY=$key`nPORT=3002" | Out-File -Encoding ascii .env

docker-compose.yml

services:
  freellmapi:
    image: ghcr.io/nexovarian-alt/freellmapi-enhanced-cn:v1.1.0
    env_file: [.env]
    environment:
      NODE_ENV: production
      PORT: 3001
      PROXY_URL: ${PROXY_URL:-}
      NODE_OPTIONS: --require=/app/bootstrap-proxy.cjs
    extra_hosts:
      - "host.docker.internal:host-gateway"
    ports:
      - "127.0.0.1:${PORT:-3002}:3001"
    volumes:
      - freellmapi-data:/app/server/data
    restart: unless-stopped
volumes:
  freellmapi-data:

启动：docker compose pull，再执行 docker compose up -d。

访问：http://localhost:3002。执行 docker compose ps，看到容器为 running 或 healthy 即安装成功。

常见错误：3002 被占用时换一个未使用端口；Docker 未运行时先打开 Docker Desktop；海外 Provider 连接失败时检查代理。

群晖 NAS 安装教程

准备条件

安装 Container Manager，在 File Station 创建 /volume1/docker/freellmapi-enhanced-cn/data。存储空间名称不同的用户请替换实际路径。

Container Manager 项目 Compose

services:
  freellmapi:
    image: ghcr.io/nexovarian-alt/freellmapi-enhanced-cn:v1.1.0
    environment:
      NODE_ENV: production
      PORT: 3001
      ENCRYPTION_KEY: "请替换为自己生成的64位十六进制密钥"
      NODE_OPTIONS: --require=/app/bootstrap-proxy.cjs
      PROXY_URL: "http://192.168.x.x:7890"
    ports:
      - "3002:3001"
    volumes:
      - /volume1/docker/freellmapi-enhanced-cn/data:/app/server/data
    restart: unless-stopped

只使用国内 Provider 时可把 PROXY_URL 留空。启动项目后访问 http://你的设备IP:3002。

首次从其他设备访问时，如果页面要求 Setup Code，请从容器启动日志中查找并填写，不要公开这个 Code。

常见错误：代理地址不能写 127.0.0.1；代理软件必须允许局域网连接；数据目录需要正确读写权限。
Linux 与通用 Docker 安装教程

准备条件

安装 Docker Engine 和 Docker Compose，确认当前用户可以运行 Docker。

准备 .env

printf "ENCRYPTION_KEY=%s\nPORT=3002\nHOST_BIND=0.0.0.0\n" "$(openssl rand -hex 32)" > .env

docker-compose.yml

services:
  freellmapi:
    image: ghcr.io/nexovarian-alt/freellmapi-enhanced-cn:v1.1.0
    env_file: [.env]
    environment:
      NODE_ENV: production
      PORT: 3001
      PROXY_URL: ${PROXY_URL:-}
      NODE_OPTIONS: --require=/app/bootstrap-proxy.cjs
    ports:
      - "${HOST_BIND:-127.0.0.1}:${PORT:-3002}:3001"
    volumes:
      - freellmapi-data:/app/server/data
    restart: unless-stopped
volumes:
  freellmapi-data:

运行 docker compose pull 和 docker compose up -d，从局域网访问 http://你的设备IP:3002。

常见错误：防火墙未允许 3002、端口冲突或 Volume 权限错误。只在可信任的家庭局域网开放访问。

代理配置说明

连接你自己已有的正常网络环境

百度千帆、阿里云百炼等国内 Provider 通常可以直接使用。Google AI Studio、Groq、OpenRouter 等海外 Provider 是否可用，取决于用户自己的网络环境。

Windows / Mac Docker Desktop

PROXY_URL=http://host.docker.internal:7890

7890 是常见 HTTP 代理端口，请以代理软件实际设置为准。

NAS / Linux

PROXY_URL=http://192.168.x.x:7890

填写运行代理软件的设备局域网 IP，并在代理软件中允许局域网连接。

本项目不提供、不宣传代理服务，也不提供绕过限制的方法；这里只说明 Docker 如何连接用户自己已有的合法网络环境。

升级与回滚

升级不需要重新添加 Provider

正确升级会保留 API Key、Provider 配置、用户配置、数据库和统计数据。

docker compose pull
docker compose up -d
升级前请确认：不要删除 Volume；不要删除数据目录；不要重新初始化数据库；必须保留原来的 ENCRYPTION_KEY。
需要回滚时怎么做

先停止当前容器并备份数据目录，把 Compose 中的镜像标签改回之前记录的版本，然后重新启动。保持 Volume 映射和 ENCRYPTION_KEY 不变。不要执行带 -v 的删除命令。

常见问题

先判断原因，再决定操作

Google 或 Groq 显示异常怎么办？

先检查代理软件是否运行、代理端口是否正确，再点击“立即检测全部 Provider”。不要因为一次失败就删除 Key。

为什么 NAS 本身能访问海外服务，但 Docker 容器不能？

容器不会自动继承 NAS 或电脑的代理。需要在 Compose 中填写 PROXY_URL；NAS 使用运行代理软件的设备局域网 IP，不能填写 127.0.0.1。

Provider 突然异常是否代表 Key 失效？

不一定。也可能是网络波动、代理未启动、上游服务临时异常或额度不足。v1.1.0 会分别显示网络、认证、额度和模型状态。

如何使用“立即检测全部 Provider”？

进入 Provider 或密钥页面，点击“立即检测全部 Provider”，等待检测结束并刷新状态。修改代理或 Key 后可以立即使用，无需等待后台周期检查。

国内 Provider 是否需要代理？

百度千帆、阿里云百炼通常可以直接使用。只有实际网络需要时才配置代理。

升级后 Key 没有显示怎么办？

检查新容器是否仍挂载原 Volume 或数据目录，以及是否继续使用原 ENCRYPTION_KEY。不要在空 Volume 中继续添加新数据。

Docker 显示 unhealthy 怎么办？

确认镜像标签是 v1.1.0，执行 docker compose pull 和 docker compose up -d。v1.1.0 已修复代理影响健康检查的问题。

忘记 ENCRYPTION_KEY 怎么办？

先查找原来的 .env、Compose、密码管理器或备份。不要生成新 Key 覆盖旧配置，否则原 Provider Key 可能无法解密。

如何备份数据？

同时备份数据 Volume 对应目录、Compose 文件和 .env。NAS 用户备份映射到 /app/server/data 的宿主机目录。备份文件不要公开上传。

如何确认正在使用 v1.1.0？

检查 Compose 的 image 标签是否为 ghcr.io/nexovarian-alt/freellmapi-enhanced-cn:v1.1.0，重新拉取后再查看容器使用的镜像。
